### 1.1 Dutch Auction (أهم حاجة ناقصة)

النوع ده فيه multiple items for auction وكل bidder يقدر يكسب واحد أو أكتر من الكمية الكلية — والسعر النهائي بيتحدد بأقل عرض فائز لكن الـ higher bidders مضمونين كامياتهم.

**إزاي نضيفه في geocore-next:**
```go
// backend/internal/auctions/model.go
// أضف للـ Auction struct:
AuctionType   string  `gorm:"default:standard" json:"auction_type"` 
// standard | dutch | reverse | buy_now_only
Quantity       int     `gorm:"default:1" json:"quantity"`
QuantityLeft   int     `json:"quantity_left"`

// Dutch Auction Logic:
// - Multiple winners allowed (up to quantity)
// - All winners pay the LOWEST winning bid
// - Track winning bids array not just current_bid
```

### 1.3 Price Plans System (نظام التسعير)

Price plan بيتربط بكل user group وبيخليك تحسب على المستخدمين في الـ group ده بطريقة معينة. كل user group ممكن يكون عنده price plan مختلف.

**أنواع التسعير في الأصلي:**
```
1. Free       — مجاني تماماً
2. Fee-based  — رسوم لكل listing
3. Subscription — اشتراك شهري/سنوي بـ X إعلان
4. Hybrid     — مجاني + upgrade options
```

**إزاي نضيفه:**
```go
// backend/internal/pricing/model.go (جديد)
type PricePlan struct {
    ID                uuid.UUID `gorm:"primaryKey"`
    Name              string
    Type              string    // free | fee | subscription
    ListingFee        float64   // رسوم لكل إعلان
    FeaturedFee       float64   // رسوم featured
    MaxListings       int       // -1 = unlimited
    MaxImagesPerListing int
    AuctionAllowed    bool
    DutchAuctionAllowed bool
    Currency          string
    BillingPeriod     string    // monthly | yearly | once
    Price             float64   // سعر الاشتراك
}

type UserGroup struct {
    ID          uuid.UUID `gorm:"primaryKey"`
    Name        string
    PricePlanID uuid.UUID
    MaxListings int
    IsDefault   bool      // الـ group الافتراضية للـ new users
}
```

**Tasks:**
- [ ] `PricePlan` + `UserGroup` models
- [ ] ربط كل user بـ user group عند التسجيل
- [ ] Admin: إنشاء/تعديل price plans
- [ ] Listing creation يتحقق من الـ price plan الخاص بالمستخدم
- [ ] Final Value Fee: عمولة من سعر البيع النهائي

### 1.4 Anonymous Listings (زي Craigslist)

الـ Anonymous Listings Addon يسمح للبائعين بنشر إعلانات بدون عملية تسجيل — العملية مشابهة لما هو متاح في مواقع مجانية كتيرة زي Craigslist و Kijiji و Gumtree.

**إزاي نضيفه:**
```go
// في listings handler:
// لو مفيش token → anonymous listing
// بيطلب: email فقط (للتحقق والتواصل)
// بيبعت email فيه رابط لإدارة الإعلان
// Admin بيعتمده قبل ما يظهر

type AnonymousListing struct {
    ListingID    uuid.UUID
    Email        string
    ManageToken  string // UUID secret لإدارة الإعلان
    ExpiresAt    time.Time
}
```

### 2.4 ZIP/Radius Search

الـ ZIP/Postal Code Search Addon أداة بحث متعددة الاستخدامات تسمح للعملاء بإجراء بحث ديناميكي بالكود البريدي في classifieds أو auctions أو directory listings.

**إزاي نضيفه:**
```go
// في listings handler List():
if radius := c.Query("radius"); radius != "" {
    lat := c.Query("lat")
    lng := c.Query("lng")
    // PostGIS query:
    // WHERE ST_DWithin(
    //   ST_MakePoint(longitude, latitude)::geography,
    //   ST_MakePoint($lng, $lat)::geography,
    //   $radius * 1000  -- km to meters
    // )
}
```

الـ PostGIS موجود في الـ DB بالفعل ✅ — بس محتاج يتفعّل في الـ search query.
